## Local Development with Docker

### Prerequisites
- Docker & Docker Compose installed
- Git repository cloned
- M-Pesa test credentials (optional for local testing)

### Setup

1. **Create .env file from templates:**
```bash
cp .env.example .env.local
# Edit .env.local with your local credentials
```

2. **Start all services:**
```bash
docker-compose up -d
```

3. **Verify services are running:**
```bash
docker ps
# Should show: postgres, backend, frontend
```

4. **Check service health:**
```bash
# Backend
curl http://localhost:3001/health

# Frontend
curl http://localhost:3000/health

# Database
docker exec trfc_postgres pg_isready -U postgres
```

5. **View logs:**
```bash
# All services
docker-compose logs -f

# Specific service
docker-compose logs -f backend
docker-compose logs -f frontend
docker-compose logs -f postgres
```

### Database Setup

The schema.sql is automatically run on first startup. To reset:

```bash
# Stop services
docker-compose down

# Remove postgres volume (data will be deleted!)
docker volume rm trfc_postgres_data

# Start fresh
docker-compose up -d
```

### Development Workflow

```bash
# Make code changes
# Services auto-reload due to volume mounts

# Run tests
docker-compose exec backend npm run test:unit
docker-compose exec frontend npm run test

# Stop services
docker-compose down

# Clean everything
docker-compose down -v
```

## Staging Deployment

### Using Railway.app

Railway provides free PostgreSQL and Node hosting with Docker support.

**Step 1: Create Railway Account**
- Go to [railway.app](https://railway.app)
- Sign up with GitHub
- Create new project

**Step 2: Connect GitHub Repository**
- Add GitHub repo to Railway
- Select this repository
- Grant necessary permissions

**Step 3: Setup Services**

**Backend Service:**
```
Name: trfc-backend
Docker File: server/Dockerfile
Port: 3000
```

**Frontend Service:**
```
Name: trfc-frontend
Docker File: client/Dockerfile
Port: 3000
```

**PostgreSQL Database:**
```
Railway → Add Service → PostgreSQL
Generate connection string
```

**Step 4: Environment Variables**

In Railway dashboard, add variables:

**Backend:**
```
NODE_ENV=staging
DATABASE_URL=[generated by Railway PostgreSQL]
JWT_SECRET=[generate strong random string]
MPESA_CONSUMER_KEY=test_key
MPESA_CONSUMER_SECRET=test_secret
MPESA_PASSKEY=test_passkey
CALLBACK_URL=https://[backend-domain]/api/payments/callback
API_BASE_URL=https://[backend-domain]
```

**Frontend:**
```
VITE_API_BASE_URL=https://[backend-domain]
```

**Step 5: Deploy**
```bash
# Push to develop branch triggers deployment
git push origin develop

# Or manually trigger in Railway dashboard
```

**Step 6: Test**
```
Backend: https://[your-staging-backend].railway.app
Frontend: https://[your-staging-frontend].railway.app
```

## Production Deployment

### Recommended: Using Railway + Vercel

#### Backend Deployment (Railway)

1. **Create Production Environment in Railway**
   - Create new project for production
   - Connect GitHub repo
   - Select main branch

2. **Configure Backend Service**
```bash
# In Railway environment variables
NODE_ENV=production
DATABASE_URL=postgresql://...
JWT_SECRET=[strong-random-string]
MPESA_CONSUMER_KEY=[production-key]
MPESA_CONSUMER_SECRET=[production-secret]
MPESA_PASSKEY=[production-passkey]
CALLBACK_URL=https://api.yourdomain.com/api/payments/callback
API_BASE_URL=https://api.yourdomain.com
```

3. **Setup Custom Domain**
   - Railway → Settings → Domains
   - Add custom domain: api.yourdomain.com
   - Update DNS CNAME records

4. **Enable SSL/HTTPS**
   - Railway auto-generates SSL certificates
   - Update .env CALLBACK_URL to HTTPS

#### Frontend Deployment (Vercel)

1. **Connect to Vercel**
   - Go to [vercel.com](https://vercel.com)
   - Import this repository
   - Select `client` as root directory

2. **Configure Build Settings**
```
Framework: Vite
Build Command: npm run build
Output Directory: dist
```

3. **Environment Variables**
```
VITE_API_BASE_URL=https://api.yourdomain.com
VITE_APP_NAME=TRFC
```

4. **Setup Custom Domain**
   - Vercel → Domains
   - Add: yourdomain.com
   - Update DNS records

#### Database Setup (Railway PostgreSQL)

1. **Create PostgreSQL Service**
   - Railway → Add Service → PostgreSQL
   - Generate connection string

2. **Initialize Schema**
```bash
# From local machine
psql [CONNECTION_STRING] < schema.sql

# Or use Railway database UI
```

3. **Backup Strategy**
```bash
# Create backup
pg_dump [CONNECTION_STRING] > backup-$(date +%Y%m%d).sql

# Restore from backup
psql [CONNECTION_STRING] < backup-20240101.sql
```

## Database Management

### Migrations

```bash
# Create migration
# Edit schema.sql

# Apply migration
docker-compose exec postgres psql -U postgres -d trfc_db -f /docker-entrypoint-initdb.d/01-schema.sql

# Verify
docker-compose exec postgres psql -U postgres -d trfc_db -c "\dt"
```

### Backups

```bash
# Backup database
docker-compose exec postgres pg_dump -U postgres trfc_db > backup.sql

# Restore database
docker-compose exec postgres psql -U postgres trfc_db < backup.sql

# Automated backup (cron job)
0 2 * * * /usr/local/bin/backup-db.sh
```

### Monitoring Database

```bash
# Connect to database
docker-compose exec postgres psql -U postgres -d trfc_db

# Useful queries
SELECT version();
SELECT * FROM users;
SELECT COUNT(*) FROM orders;
SELECT COUNT(*) FROM payment_callbacks;
```

## Troubleshooting

### Common Issues

#### 1. Services won't start
```bash
# Check logs
docker-compose logs

# Common causes:
# - Port already in use: docker ps -a
# - Insufficient disk space: docker system df
# - Missing .env file: cp .env.example .env.local

# Solution
docker-compose down
docker system prune
docker-compose up -d
```

#### 2. Database connection failed
```bash
# Test connection
docker-compose exec postgres psql -U postgres

# If fails:
# - Check DATABASE_URL in .env
# - Verify postgres service is healthy: docker-compose ps
# - Check logs: docker-compose logs postgres
```

#### 3. Backend crashes
```bash
# Check logs
docker-compose logs backend

# Common causes:
# - Missing environment variables
# - Database not ready
# - Port already in use

# Solution
docker-compose restart backend
```

#### 4. Frontend not loading
```bash
# Check logs
docker-compose logs frontend

# Common causes:
# - API_BASE_URL misconfigured
# - Build failed
# - Nginx config issue

# Solution
docker-compose restart frontend
```

#### 5. Payment API errors
```bash
# Check backend logs
docker-compose logs backend | grep -i payment

# Verify M-Pesa credentials in .env
# Check CALLBACK_URL is reachable
curl -I [CALLBACK_URL]
```

### Health Checks

```bash
# Backend
curl http://localhost:3001/health

# Frontend
curl http://localhost:3000/health

# Database
docker-compose exec postgres pg_isready -U postgres
```

### Performance Optimization

```bash
# Reduce image size
docker image prune -a

# Check resource usage
docker stats

# Increase database connections if needed
# Edit docker-compose.yml postgres service

# Enable query logging (development only)
# psql > set log_statement = 'all';
```

## Security Checklist

- ✅ Use HTTPS in production
- ✅ Store secrets in GitHub Secrets / Railway environment
- ✅ Never commit .env files
- ✅ Use strong JWT_SECRET (min 32 characters)
- ✅ Enable firewall rules
- ✅ Regular database backups
- ✅ Monitor logs for suspicious activity
- ✅ Keep dependencies updated
- ✅ Use non-root user in containers
- ✅ Enable rate limiting on API
